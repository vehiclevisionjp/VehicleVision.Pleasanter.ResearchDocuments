## 調査目的

プリザンターの拡張計算式（`FormulaServerScriptUtilities.cs`）に実装されている関数を Excel の標準関数と比較し、不足している関数を体系的に洗い出す。これにより、今後の機能拡張の優先度検討に必要な情報を整理する。

## 実装上の制約

拡張計算式は 1 レコード内の項目値のみを参照して計算する設計のため、以下のカテゴリの関数は設計上の制約がある。

| 制約の種類             | 影響を受ける関数例                     | 説明                                                                     |
| ---------------------- | -------------------------------------- | ------------------------------------------------------------------------ |
| セル範囲の概念がない   | VLOOKUP, HLOOKUP, INDEX, MATCH, OFFSET | 拡張計算式は単一レコードの項目値を対象とし、複数レコードの参照はできない |
| シートの概念がない     | SHEET, INDIRECT, ROW, COLUMN           | プリザンターにはシート/セル座標の概念がない                              |
| 条件付き範囲集計の制限 | SUMIF, COUNTIF, AVERAGEIF 等           | 複数レコードの集計は拡張計算式ではなくサーバスクリプトや API で対応する  |

| 設計上の制約           | 検索・参照関数（VLOOKUP 等）はセル範囲の概念がないため適用困難                   |
