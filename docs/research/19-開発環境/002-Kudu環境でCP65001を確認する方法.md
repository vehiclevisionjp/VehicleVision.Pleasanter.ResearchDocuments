## 調査目的

Azure App Service の Kudu 環境上で CodeDefiner 等を実行する際、日本語を含むログや出力が文字化けする問題が発生する。CP65001（UTF-8）をコードページとして設定できるか、また設定方法を明確にする。

---

## 背景

Windows のデフォルトコードページは環境によって異なる（例: CP932 = Shift_JIS、CP437 = OEM 英語）。日本語を正しく表示・処理するには CP65001（UTF-8）に切り替える必要がある。

Azure App Service の Kudu はブラウザからアクセスできる Web ベースのコンソールであり、通常の Windows CMD/PowerShell と同等の操作が可能。

---

## 確認手順

### 方法1：Kudu Debug Console（CMD）で直接確認

Azure ポータル → App Service → **高度なツール（Kudu）** → **Debug Console > CMD** を開く。

**現在のコードページを確認する：**

```cmd
chcp
```

実行例：

```
Active code page: 437
```

**CP65001 に切り替える：**

```cmd
chcp 65001
```

成功した場合の出力：

```
Active code page: 65001
```

エラーが出なければ CP65001 は使用可能。

---

### 方法2：PowerShell Console で確認

Kudu の **Debug Console > PowerShell** で実行する。

```powershell
# 切り替え前のエンコーディングを確認
[Console]::OutputEncoding

# CP65001 に切り替え
chcp 65001

# 切り替え後のエンコーディングを確認
[Console]::OutputEncoding
```

切り替え前後で `BodyName` が `utf-8` に変わることを確認できる。

---

### 方法3：日本語出力テスト

```cmd
chcp 65001 && echo テスト
```

文字化けせずに「テスト」と表示されれば CP65001 が正常に機能している。

---

## 実用的な使い方

### 起動バッチに組み込む

Kudu セッションは終了するとコードページがリセットされる。コマンド実行前に毎回設定が必要なため、バッチファイルの先頭に記述しておく。

```cmd
@echo off
chcp 65001 > nul
```

> `> nul` を付けることで「Active code page: 65001」の余分な出力を抑制できる。

### CodeDefiner 実行例

```cmd
@echo off
chcp 65001 > nul
dotnet Implem.CodeDefiner.dll rds /p "Implem.Pleasanter\App_Data\Parameters"
```

## 注意事項

| 項目                 | 内容                                                                                                 |
| -------------------- | ---------------------------------------------------------------------------------------------------- |
| セッション間の永続化 | Kudu セッションを閉じると設定がリセットされる。バッチファイルへの組み込みが必要                      |
| PowerShell との違い  | `chcp 65001` を実行しても PowerShell の `[Console]::OutputEncoding` は別途変更が必要になる場合がある |
| Linux App Service    | Linux ベースの App Service はデフォルト UTF-8 のため本手順は不要                                     |

---

## 結論

| 確認項目             | 結果                                                                 |
| -------------------- | -------------------------------------------------------------------- |
| CP65001 の使用可否   | Azure App Service（Windows）の Kudu では使用可能                     |
| 確認コマンド         | `chcp 65001`（エラーなし＝使用可能）                                 |
| 日本語出力の文字化け | `chcp 65001` 設定後は解消される                                      |
| 設定の永続化         | セッション終了でリセットされるため、バッチファイルへの組み込みが必要 |
